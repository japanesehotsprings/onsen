全15項目 実装ロードマップ

既存データへの追記から始め、新ページ → 特集コンテンツの順に段階的に強化する

⚡ フェーズ1 — データ追記（すぐできる・効果大）

既存の hotels.js / onsen.js にフィールドを追加するだけ。新しいページ不要。

追加するデータフィールドと表示例
A2 — 効能バッジ

onsen.js に effects: ['美肌', '疲労回復', '神経痛'] フィールドを追加。温泉地・宿カードにバッジ表示。検索・フィルターにも活用。

B1 — 源泉かけ流し情報

hotels.js に source: '源泉かけ流し'（or '加水あり' / '循環式'）を追加。宿詳細ページに明示表示。

B2 — 日帰り入浴情報

onsen.js に dayTrip: { available: true, price: '800円〜', hours: '10:00-15:00' } を追加。温泉地詳細ページに掲載。

B3 — こだわりバッジ（宿の特徴）

hotels.js に features: ['露天付き客室', '貸切風呂', '展望風呂', '24時間入浴可'] を追加。宿カードにアイコン付きで表示。

B6 — アクセス情報

onsen.js に access: { nearest_station: '草津温泉バスターミナル', by_train: '新幹線+バス約2時間', parking: true } を追加。

🏗 フェーズ2 — 新ページ・コンテンツ追加（中規模）

新しいデータファイルとページコンポーネントを作成する。

新規ページ・データ
A1 — 泉質詳細ページ

新規 springTypes.js データファイル + 泉質詳細ページ。URL例: /spring-type/sulfur。効能・特徴・代表温泉地一覧を掲載。

A3 — 泉質別おすすめ一覧

「硫黄泉の名湯一覧」など、泉質でフィルターした温泉地リストページ。既存データのフィルタリングで実装可能。

A4 — 飲泉・足湯情報 / B4 — 食事情報 / B5 — 歴史・由来

各データファイルへのフィールド追加と、詳細ページへの表示追加。
onsen.js: history, drinkable, footbath
hotels.js: meals（夕食・朝食の特徴）

📰 フェーズ3 — 特集・編集コンテンツ（大きな差別化）

新しいコンテンツ種別の設計が必要。最も差別化になる。

特集コンテンツ
C1 — 季節別特集ページ

新規 specials.js（特集データ）+ 特集一覧・詳細ページ。「雪見露天」「紅葉×温泉」「夏の避暑」など。既存の温泉地データをタグで紐づけ。

C2 — シーン別おすすめ

「一人旅」「カップル」「家族」「女子旅」などのシーンタグを宿データに追加 + シーン別フィルターページ。

C3 — 初めての温泉旅行ガイド

新規ガイドページ。温泉の入り方・マナー・持ち物・泉質の選び方など。SEO流入の入口として機能。

C4 — エリア別モデルコース

新規 courses.js + モデルコースページ。既存のグルメ・観光スポットデータを組み合わせて「1泊2日プラン」を構成。

C5 — 特集記事・ランキング

新規 articles.js + 記事一覧・詳細ページ。「関東から日帰りできる温泉10選」などのキュレーション記事。SNSシェア・ブックマーク誘導。
