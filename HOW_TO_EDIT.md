# メディア「日本温泉」の編集・運用マニュアル

このドキュメントは、メディアサイト「日本温泉」にご自身でデータやコンテンツを追加・修正するためのガイドです。

---

## 📂 プロジェクトの主要なファイル構成

編集する機会が多いのは、主に `src/` フォルダ内のファイルです。

### 1. データファイル（情報を追加・修正する場所）
- **`src/data/onsen.js`** … 全国47都道府県の「温泉地」のデータ
- **`src/data/hotels.js`** … 各温泉地にある「温泉宿」のデータ
- **`src/data/prefectures.js`** … 「都道府県・地域」の基本データ

### 2. ページコンポーネント（画面の見た目や文章を変える場所）
- **`src/pages/home.js`** … トップページ（「おすすめ温泉宿」「注目の温泉地」など）
- **`src/pages/prefectureDetail.js`** … 各都道府県のページ（その県の温泉一覧）
- **`src/pages/onsenDetail.js`** … 温泉地ごとの詳細ページ（温泉の紹介と、周辺の宿一覧）
- **`src/pages/hotelDetail.js`** … 各温泉宿の詳細ページ（宿の基本情報や予約サイトへのボタン）
- **`src/pages/searchResults.js`** … 検索結果ページ

### 3. デザイン・スタイル
- **`src/style.css`** … サイト全体の色使い、余白、文字サイズ、カードのデザインなど

---

## ♨️ データの追加・修正方法

最もよく行う「温泉地の追加」と「温泉宿の追加」の手順です。

### 1. 新しい「温泉地」を追加する
`src/data/onsen.js` を開きます。  
`const onsenList = [ ... ];` の配列（リスト）の中に、新しい温泉地データを波括弧 `{}` で追加します。

**テンプレート例:**
```javascript
{
    id: 'o-gunma-3',           // 重複しない固有のID（例: o-県名-連番）
    prefecture: 'gunma',       // 所属する都道府県のID（prefectures.js参照）
    name: '四万温泉',          // 温泉地の名前
    description: 'ノスタルジックな雰囲気が漂う、国民保養温泉地第一号。', // 説明文
    spring_type: '塩化物・硫酸塩泉', // 泉質
    category: '美肌の湯',      // カテゴリ（バッジに表示されます）
    image: 'https://images.unsplash.com/photo-...', // 画像URL
    tags: ['千と千尋のモデル', '飲泉', '昭和レトロ'] // 検索用＆表示用のタグ（複数可）
}
```

### 2. 新しい「温泉宿」を追加する
`src/data/hotels.js` を開きます。  
`const hotelList = [ ... ];` の中に、新しい宿のデータを追加します。追加すると、自動で該当する温泉地ページ（onsenDetail.js）にその宿が表示されます。

**テンプレート例:**
```javascript
{
    id: 'h-gunma-3-1',         // 宿の固有ID（例: h-県名-温泉連番-宿連番）
    onsen_id: 'o-gunma-3',     // 所属する「温泉地」のID（上の例だと四万温泉）
    name: '積善館',            // 宿の名前（※予約サイト検索に使われます）
    prefecture: 'gunma',       // 都道府県ID
    type: '重要文化財の宿',    // 宿の種類やキャッチコピー
    description: '日本最古の木造湯宿建築。大正ロマンあふれる「元禄の湯」が有名。', // 宿の説明
    price: '2食付き 15,000円〜', // 参考料金
    image: 'https://images.unsplash.com/photo-...', // 画像URL
    tags: ['国指定重要文化財', '元禄の湯', '歴史的建築'] // 特徴タグ
}
```
> **💡 ヒント:** `hotelDetail.js` の予約サイトへのボタンは、ここの `name`（「積善館」など）を元にして自動的に検索URLを作るようになっています。予約サイトで検索しやすい正式名称を入れるのがおすすめです。

## 🎨 画面構成やテキストの変更方法

トップページなどの「見出しテキスト」や「表示ロジック」を変えたい場合の例です。

### 1. トップページのテキストを変える
`src/pages/home.js` を開きます。  
`app.innerHTML = ...` の中に書いてあるHTMLテキストが実際の画面に表示されます。
- `hero-title` や `hero-subtitle` の中身を書き換えると、一番上の大きな文字が変わります。
- `<h2 class="section-title">おすすめ温泉宿</h2>` などの文字を書き換えれば、見出しも自由に直せます。

### 2. おすすめ宿・注目温泉地の「取得件数」を変える
同じく `src/pages/home.js` の最初の方に以下のようなコードがあります。
```javascript
const pickupOnsen = getPickupOnsen(); // ここで4件取得している
const featuredHotels = getRandomHotels(8); // ★8件取得している
```
例えばトップページに宿を12件表示したい場合は、`getRandomHotels(12)` に数字を変えるだけで反映されます。

### 3. 色合いを変える
`src/style.css` の一番上にある `:root` の数値を変更します。
```css
:root {
  --color-primary: #d4845a;       /* メインカラー（オレンジ・茶系） */
  --color-primary-light: #e6b299; /* 薄いメインカラー */
  --color-accent: #7dbfb5;        /* アクセントカラー（青緑系） */
  --color-bg: #110e0b;            /* 背景色ベース（ダークモード） */
  ...
}
```
ここを変えるだけで、サイト全体のボタンや文字の色が一括で変わります。

## 💻 ローカルでの確認手順

修正した内容を確認するには、ターミナルで以下のコマンドを実行します。

1. **開発サーバーを立ち上げる**
   ```bash
   npm run dev
   ```
   ターミナルに表示されるURL（ `http://localhost:5173/` など）にブラウザでアクセスすると、修正した内容が即座に反映される状態で確認できます。

2. **本番公開用にビルドする**
   サーバー（VercelやNetlify、GitHub Pagesなど）にファイルをアップロードする準備をするコマンドです。
   ```bash
   npm run build
   ```
   エラーなく `dist/` フォルダが作成・更新されれば成功です。
